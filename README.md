# actividad-git-github

## Control de versiones

Antes de conocer Git, normalmente manejaba las diferentes versiones de mis trabajos creando copias del mismo archivo pero con nombre distinto, por ejemplo “trabajo-final”, “trabajo-final2” y asi sucesivamente hasta tener el trabajo final final.

El problema de este método es que muchas veces terminaba confundido sobre qué cambios había hecho en cada archivo.

Creo que Git puede resolver estos problemas porque permite guardar un historial completo de cambios, volver a versiones anteriores y mantener todo mucho más organizado.
## Commits significativos

Creo que hacer commits con mensajes descriptivos es importante para poder recordar que cambios hice y en que momento del proyecto. También ayuda a organizarse mejor y facilita encontrar errores.
Además, si varias personas trabajan en el mismo proyecto, los mensajes claros ayudan a que todos entiendan qué hizo cada integrante.
